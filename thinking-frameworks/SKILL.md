# 25种思维框架工具

> "你的智慧取决于你的思维工具。" — Scott H. Young

## 概述

本skill提供25种来自不同职业的思维工具。每种工具代表一个领域的核心思维方式，可跨领域应用于问题分析、决策制定和创造性解决方案。

## 何时使用

- 分析复杂问题时
- 需要多角度思考时
- 做重要决策时
- 寻找创造性解决方案时
- 诊断问题根因时
- 设计系统或产品时
- 理解他人行为动机时

## 25种思维工具

### 一、创造与设计类

| 工具 | 核心思维 | 应用场景 |
|------|---------|---------|
| **艺术家** | 优先创意和独特性，突破约束 | 需要创新、突破常规时 |
| **建筑师** | 创建缩放模型预见结果 | 规划大型项目、原型设计 |
| **设计师** | 让物品自然传达其用途 | UI/UX设计、产品设计 |
| **小说家** | 将信息结构化为连贯叙事 | 写报告、讲故事、说服他人 |

## 使用方法

### 单一工具应用

当问题明确属于某一领域时，选择最相关的工具：

```
问题：用户流失率上升
适用工具：医生思维（诊断根因）→ 找出流失的真正原因
```

### 多工具组合

复杂问题往往需要组合多种思维：

```
问题：设计新产品
组合工具：
1. 企业家（快速原型验证）
2. 设计师（让功能自明）
3. 销售员（洞察未表达需求）
4. 经济学家（设计激励机制）
```

### 视角切换清单

当卡住时，依次尝试以下视角：

1. **拆解视角**（水管工）：这个问题能拆成哪些部分？
2. **根因视角**（医生）：表面症状背后的真正原因是什么？
3. **激励视角**（经济学家）：各方的激励机制是什么？
4. **感知视角**（政治家）：这会被如何解读？
5. **底层视角**（黑客）：表象之下的真实机制是什么？
6. **验证视角**（科学家）：我的假设能被检验吗？
7. **简化视角**（程序员）：有重复模式可以简化吗？

## 工作流

### Phase 1: 问题定义
1. 明确陈述问题
2. 识别约束条件
3. 确定成功标准

### Phase 2: 工具选择
1. 判断问题类型（创造/分析/系统/诊断/战略/执行）
2. 选择1-3个最相关的思维工具
3. 记录选择理由

### Phase 3: 应用分析
1. 用选定工具依次分析问题
2. 记录每种视角的洞察
3. 综合不同视角的结论

### Phase 4: 方案形成
1. 基于分析形成候选方案
2. 用验证类工具（科学家/数学家/记者）检验方案
3. 选择最佳方案

## 核心原则

1. **没有万能工具**：不同问题需要不同工具
2. **组合胜于单一**：复杂问题需要多工具组合
3. **视角决定洞察**：换个视角可能看到完全不同的解
4. **工具需要练习**：熟练掌握需要反复应用
5. **元认知优先**：先思考如何思考，再思考问题本身
